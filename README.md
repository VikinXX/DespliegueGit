# División del Documento: Capítulo 6

## Parte 1: Fundamentos de Documentación y Javadoc

**Participantes Principales:**
- *Alvaro Ortiz Villarejo*

**Descripción de lo que hacen:**
- El equipo de desarrollo comprende la importancia de la documentación y del control de versiones para la colaboración efectiva.
- Se introduce el uso de Javadoc como herramienta principal para automatizar la documentación de aplicaciones Java.
- El equipo aprende a documentar clases (usando etiquetas como @author, @deprecated, @version) y métodos (usando @param, @return, @throws).
- Se detallan los pasos para generar la documentación final utilizando Javadoc.

## Parte 2: Herramientas de Documentación Automatizada (PHP y Versátil)

**Participantes Principales:**
- *Equipo de desarrollo (PHP y multiplataforma)*
- *phpDocumentor*
- *Doxygen*

**Descripción de lo que hacen:**
- El equipo explora herramientas más allá de Java para documentar código en PHP y otros lenguajes.
- Se utiliza phpDocumentor para documentar variables globales, clases, funciones y métodos en proyectos PHP.
- El equipo configura phpDocumentor en Netbeans, teniendo en cuenta requisitos como XAMPP y la versión estable.
- Se emplea Doxygen como una solución versátil para documentar código en C++, Java, Python y otros lenguajes.
- El equipo configura proyectos en Doxygen, elige el lenguaje de programación y el formato de salida (por ejemplo, HTML).

## Parte 3: Estándares de Documentación Manual y Plantillas

**Participantes Principales:**
- *Equipo de desarrollo*
- *Netbeans (plantillas/templates)*

**Descripción de lo que hacen:**
- El equipo establece normas claras de nomenclatura y documentación interna del código (comentarios en línea, bloque y especiales para Javadoc).
- Se define qué hay que documentar en clases y métodos (descripción, autor, versión, tipo, parámetros, salida).
- Los desarrolladores utilizan y crean plantillas en Netbeans para agilizar la inclusión de comentarios estándar (autor, versión) en el código.
- El equipo aprende a modificar plantillas usando variables automáticas como `${user}`, `${date}`, `${name}`.

## Parte 4: Herramientas Colaborativas y Teoría de Control de Versiones

**Participantes Principales:**
- *Equipo de desarrollo*
- *Slack*
- *Microsoft Office 365*
- *Sistemas de Control de Versiones (SCV): Git, Github, Subversion, etc.*

**Descripción de lo que hacen:**
- El equipo utiliza Slack y Microsoft Office 365 para comunicarse y colaborar (creación de canales, compartir archivos, integración de apps, colaboración en documentos).
- Los participantes gestionan archivos y documentación usando OneDrive y otras aplicaciones de Office 365.
- Se estudia y comprende la teoría de control de versiones: cómo funcionan los repositorios, ramas, conflictos, revisiones y confirmaciones ("commit").
- Se analizan las distintas arquitecturas de SCV: local, centralizada y distribuida.

## Parte 5: Implementación Práctica de Git, Operaciones Avanzadas y Seguridad

**Participantes Principales:**
- *Equipo de desarrollo*
- *Git*
- *Github*
- *Netbeans (como cliente)*

**Descripción de lo que hacen:**
- El equipo configura Git y Github para establecer repositorios centralizados y distribuidos, siguiendo los pasos de inicialización, clonación y push.
- Los desarrolladores ejecutan operaciones avanzadas con Git: comparar versiones (DIFF), revertir cambios, gestionar ramas, ejecutar merges/patches y resolver conflictos.
- Se configuran protocolos de acceso y seguridad en Netbeans (SSH y HTTPS) para proteger el código fuente.
- El equipo visualiza el historial de los repositorios, filtra cambios por mensaje, autor, rama o fecha, y estudia las diferencias entre archivos de distintas ramas.
